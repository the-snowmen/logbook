# logbook — {Project Name}

A gitignored, file-based dev journal: the decisions, phases, fixes, and handoffs that capture *how and
why* this project is built, so they survive context resets. One fact per file, cross-linked with
`[[name]]`. Companion to `CLAUDE.md` (the locked rules/architecture).

## How to use
Mostly automatic: each session opens with a read-back digest (newest handoff → in-progress phases →
open items), and decisions, fixes, phases, and session summaries are recorded as work happens per your
tracking mode (auto / suggest / manual — `~/.config/logbook/config.json`). This index updates as
entries land.
- Deeper start-of-session read-back on demand: `/logbook:logbook-start`.
- Manual entries anytime: `/logbook:logbook-decision`, `/logbook:logbook-troubleshooting`,
  `/logbook:logbook-mark-phase`, `/logbook:logbook-end-session`, `/logbook:logbook-handoff` (newest wins).

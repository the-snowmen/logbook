# logbook — {Project Name}

A gitignored, file-based dev journal: the decisions, phases, fixes, and handoffs that capture *how and
why* this project is built, so they survive context resets. One fact per file, cross-linked with
`[[name]]`. Companion to `CLAUDE.md` (the locked rules/architecture).

## How to use
1. **Start a session** — `/logbook:logbook-start` (reads CLAUDE.md → newest handoff → phases).
2. **During** — real decision → `/logbook:logbook-decision`; problem solved → `/logbook:logbook-troubleshooting`.
3. **Phase boundary** — `/logbook:logbook-mark-phase` to record the outcome; then stop, report, await go-ahead.
4. **End a session** — `/logbook:logbook-end-session` + a fresh `/logbook:logbook-handoff` (newest wins). Update this index.
